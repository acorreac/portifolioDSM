<h2>Meus Projetos</h2>

<h3>Equipe Spicecorp</h3>
<h3>Spicebank</h3>

<p align="justify">Parceiro Acadêmico: Faculdade de Tecnologia de São José dos Campos</p>

<p align="center"><img src="https://github.com/acorreac/portifolioDSM/blob/main/spicecorp.png" widht="20%"></img>

<p align="justify">O objetivo do projeto foi criar um sistema para gerenciamento de transações em um banco digital fictício, visando monitorar e controlar operações bancárias de um banco digital, reduzindo as interações presenciais.</p>
<br>
<h3>Tecnologias Utilizadas</h3>
<p align="justify"><a href="https://flask.palletsprojects.com/en/3.0.x/">Flask:</a> framework utilizado para o desenvolvimento web(front-end).</p>
<p align="justify"><a href="https://developer.mozilla.org/pt-BR/docs/Web/CSS">CSS:</a> utilizado para o a estilização do desenvolvimento web(front-end).</p>
<p align="justify"><a href="https://www.python.org/">Python:</a> linguagem de programação para o desenvolvimento back-end.</p>
<p align="justify"><a href="https://mariadb.org/">MariaDB:</a> utilizado para o armazenamento dos dados dos itens cadastrados.</p>
<p align="justify"><a href="https://www.figma.com/">Figma:</a> para o desenvolvimento e prototipação das wireframes.</p>
<br>
<h3>Contribuições Pessoais</h3>
<p align="justify">Desempenhei o papel de Master e Develop. Como Master realizando a organização da equipe e das entregas, observando e pontuando o progresso do burndown e das tarefas. Em develop atuei como FullStack, contribuindo nas decisões de design, funcionalidades do frontend e integração do frontend com o backend. No backend estive presente em todo o desenvolvimento lógico e funcional, modelagem dos dados, integração entre o banco de dados, backend e frontend. Sendo as atividades desempenhadas:</p>
<ul>
  <li>Modelagem do banco de dados</li>
  <li>Desenvolvimento das funcionalidades Cliente</li>
  <li>Desenvolvimento das funcionalidades Gerente Geral</li>
  <li>Desenvolvimento das funcionalidades Gerente de Agência</li>
  <li>Desenvolvimento da lógica de funcionalidades do cliente e gerente</li>
  <li>Integrações dos sistemas</li>
  <li>Arquitetura da aplicação</li>
</ul>
<br>
<h3>Lições Aprendidas</h3>
<p align="justify">Lidar com pessoas em diferentes ocasiões, organizar equipe de forma a garantir as entregas, ter visão macro do projeto e resiliência.</p>

<h3>Equipe Sabiá</h3>
<h3>Sistema de Inspeção de Entrada para o Controle de Recebimento de Grãos em uma Agroindústria</h3>

<p align="justify">Parceiro Acadêmico: <a>Jaia Software</a></p>
<p align="center"><img src="https://github.com/acorreac/portifolioDSM/blob/main/sabia.png" widht="20%"></img>

<p align="justify">Consistiu no desenvolvimento de um sistema que fosse possível inspecionar a entrada para o controle de recebimento de grãos em uma agroindústria, através do processo de inserção de produto, fornecedor, pedido e nota fiscal do pedido no sistema, para posteriormente ser ppossível realizar a análise quantitativa e qualitativa do produto recebido, bem como seu laudo após tais inspeções do produto.</p>
<br>
<h3>Tecnologias Utilizadas </h3>
<p align="justify"><a href="https://react.dev/">React:</a> framework utilizado para o desenvolvimento web(front-end).</p>
<p align="justify"><a href="https://developer.mozilla.org/pt-BR/docs/Web/HTML">HTML:</a> uma linguagem de marcação utilizada na construção das páginas Web.</p>
<p align="justify"><a href="https://developer.mozilla.org/pt-BR/docs/Web/CSS">CSS:</a> utilizado para o a estilização do desenvolvimento web(front-end).</p>
<p align="justify"><a href="https://www.typescriptlang.org/">TypeSript:</a> linguagem de programação para o desenvolvimento back-end.</p>
<p align="justify"><a href="https://nodejs.org/en">Node:</a> software de código aberto, multiplataforma, baseado no interpretador V8 do Google e que permite a execução de códigos JavaScript fora de um navegador web e foi utilizado para o desenvolvimento front-end.</p>
<p align="justify"><a href="https://www.mysql.com/">MySQL:</a> utilizado para o gerenciamento e armazenamento dos dados.</p>
<p align="justify"><a href="https://www.figma.com/">Figma:</a> para o desenvolvimento e prototipação das wireframes.</p>
<br>
<h3>Contribuições Pessoais </h3>
<p align="justify">Desempenhei o papel de Develop, realizando as tarefas do backend, como o crud(create, update, read e delete) da análise qualitativa do projeto e update dos usuários do sistema, e realizei correções dos casos de testes. Sendo as atividades desempenhadas:</p>
<ul>
  <li>Create de requisitos para a análise qualitativa.</li>
  <li>Upadte de requisitos para a análise qualitativa.</li>
  <li>read de requisitos para a análise qualitativa.</li>
  <li>Delete de requisitos para a análise qualitativa.</li>
  <li>Update de usuários</li>
  <li>Casos de testes e correções</li>
</ul>
<br>
<h3>Lições Aprendidas </h3>
<p align="justify">Pude estar em uma equipe mais colaborativa, na qual me fez evoluir no meu papel de contribuinte. Devido alguns imprevisto por parte do cliente, podemos aprende na prática a administração do tempo e a rápida adaptações as mudanças do projeto.</p>

<h3>Equipe Beelieve</h3>
<h3>Software de Gestão de Times de Projetos Complexos</h3>
<h4>3º Semestre • 2/2023 • <a href="https://github.com/Equipe-Beelieve/Equipe-API-3DSM-Beelieve/tree/main">Repositório Github</a></h4>
  
<p align="justify">Parceiro Acadêmico: <a>SIATT</a></p>

<p align="center"><img src="https://github.com/acorreac/portifolioDSM/blob/main/beelieve.png" widht="20%"></img>

<p align="justify">O desafio proposto era desenvolver um "Software para Gestão de Times de Projetos Complexos" que automatize todo o trabalho de gerenciamento de equipes integradas. Esse software estaria acessível por meio de um portal na intranet da empresa, facilitando o acompanhamento e a gestão de projetos complexos que estão em desenvolvimento..</p>
<br>
<h3>Tecnologias Utilizadas  </h3>
<p align="justify"><a href="https://www.java.com/pt-BR/">Java SE 17:</a> linguagem para desenvolvimento da aplicação para back-end.</p>
<p align="justify"><a href="https://spring.io/">Spring Boot:</a> framework para Java utilizada para facilitar a configuração e o gerenciamento das dependências do projeto.</p>
<p align="justify"><a href="https://www.javascript.com/">JavaScript:</a> linguagem para desenvolvimento da aplicação para front-end.</p>
<p align="justify"><a href="https://react.dev/">React:</a> framework utilizado para o desenvolvimento web(front-end).</p>
<p align="justify"><a href="https://developer.mozilla.org/pt-BR/docs/Web/CSS">CSS:</a> utilizado para o a estilização do desenvolvimento web(front-end).</p>
<p align="justify"><a href="https://tailwindcss.com/">Tailwind:</a> utilizado junto com o css para o a estilização do desenvolvimento web(front-end).</p>
<p align="justify"><a href="https://www.mysql.com/">MySQL:</a> SGBD para desenvolvimento do Banco de Dados.</p>
<p align="justify"><a href="https://www.mongodb.com/">MongoDB:</a> utilizado para armazenar os dados do cronograma dos projetos.</p>
<br>
<h3>Contribuições Pessoais  </h3>
<p align="justify">Desempenhei o papel de develop backen, contribuindo nas tarefas de crud do projeto, criação de tarefas do projeto, crud dos usuarios, e casos de testes. Sendo as atividades desempenhadas:</p>
<ul>
  <li>Delete de projeto.</li>
  <li>Criação de tarefas.</li>
  <li>Estrutura DTO dos usuários.</li>
  <li>Criação dos casos de testes do sistema.</li>
</ul>
<br>
<h3>Lições Aprendidas</h3>
<p align="justify">Tive o meu primeiro contato com Spring Boot, e com a linguagem Java, na qual pude ter melhor entendimento sobre SOLID e padrões de código, micro-serviços, niveis de maturidade e restfull.</p>
